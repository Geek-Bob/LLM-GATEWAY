# 错误处理

## 错误类型体系
- 验证错误（ValidationError）：输入不合法，返回 400 级状态码
- 业务错误（BusinessError）：业务规则不允许，返回明确业务提示
- 系统错误（SystemError）：基础设施故障（DB 连接断开、文件写入失败），返回 500 级
- 禁止用一个通用 Error 类型处理所有错误

## 错误传播规则
- 数据层：抛出系统错误（数据库操作失败），不抛业务错误
- 业务层：抛出业务错误（规则不允许），不处理系统错误
- 接口层：捕获所有错误，映射为统一格式返回
- 错误从底层向顶层传播，每层添加自己层的上下文信息

## 错误上下文
- 每次 throw 必须携带：操作名 + 关键参数
- 错误消息格式：`Failed to {操作} {对象}: {原因}`
- 禁止抛出无上下文的错误（如 `throw new Error('error')`）

## IPC 错误映射
- IPC handler 捕获错误后，将错误信息序列化为可读字符串返回给渲染进程
- 业务错误返回用户可见消息
- 系统错误返回通用提示，详细信息记录到日志

## 代理错误映射
- 上游返回的 HTTP 错误透传给客户端
- 网络连接失败返回 502，附带错误描述
- 超时返回 504，附带超时时长

## 禁止
- 空 catch 块（`catch {}`）
- 只打印不处理（`catch(e) { console.log(e) }`）
- 吞没错误（`.catch(() => null)`）
- 将系统错误的详细堆栈暴露给用户
- 在循环中抛出错误但不中断循环（要么全部成功，要么全部回滚）
