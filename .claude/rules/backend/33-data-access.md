# 数据访问

## 查询抽象
- 业务层通过 service 调用数据操作，不直接写 SQL
- 数据层提供函数式接口（如 `findProviders()`、`insertApiKey()`）
- 复杂查询在数据层封装，业务层只传入查询条件

## 连接管理
- 数据库连接在应用启动时初始化，全局共享
- 连接生命周期由入口层管理（启动时打开，退出时关闭）
- 禁止在业务逻辑中创建新的数据库连接

## Schema 管理
- 表结构定义集中在 schema 文件中
- 新增表或字段必须在 schema 文件中声明，禁止运行时 ALTER
- 字段命名使用 snake_case（数据库层），应用层通过映射转换

## 事务边界
- 需要原子操作的多步写入必须在事务中完成
- 事务在数据层开启和提交，业务层不感知事务细节
- 事务失败必须回滚，禁止部分提交

## NDJSON 日志存储
- 请求日志使用 NDJSON 格式，每条请求一行 JSON
- 文件轮转：单文件 500 行，最多 20 文件（10000 条上限）
- 元数据（entryCounter、currentFileNumber）记录在 logs-meta.json

## 禁止
- 业务层直接编写 SQL 语句
- 数据层函数包含业务规则判断
- 在循环中逐条插入（使用批量操作）
- 日志文件使用 SQLite 表存储（NDJSON 流式写入无锁竞争）
- 数据库连接对象泄漏（未在退出时关闭）
