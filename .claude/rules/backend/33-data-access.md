# 数据访问

## 查询抽象
- 数据层提供函数式接口（如 `findProviders()`、`insertApiKey()`），封装所有 SQL 操作
- 业务层通过工厂参数注入的 db 对象调用数据层函数，禁止绕过数据层直接编写 SQL

## 连接管理
- 业务层通过 service 工厂函数参数获取 db 实例，禁止导入 `db/connection.ts` 的 `getDb()`
- 禁止在业务逻辑中创建新的数据库连接（生命周期由入口层管理）

## Schema 管理
- 表结构定义集中在 `db/schema.ts`，新增表或字段必须在此声明，禁止运行时 ALTER
- 字段命名使用 snake_case（数据库层），应用层通过映射转换

## 事务边界
- 需要原子操作的多步写入必须在事务中完成，事务在数据层开启和提交
- 事务失败必须回滚，禁止部分提交

## NDJSON 日志存储
- 请求日志以 NDJSON 格式存储（每条一行 JSON），不使用 SQLite 表
- 文件轮转：单文件 500 行，最多 20 文件（10000 条上限）
- 元数据（entryCounter、currentFileNumber）记录在 logs-meta.json

## 禁止
- 业务层导入 `db/` 目录下的任何文件（数据操作通过工厂参数注入的 db 对象调用）
- 数据层函数包含业务规则判断（如权限校验、条件过滤逻辑）
- 在循环中逐条插入（使用批量操作）
- 数据库连接对象泄漏（未在退出时关闭）
